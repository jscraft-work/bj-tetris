# 06-sound-and-vfx: 사운드/효과 기획

## 1. 목표

- 핵심 게임 조작에 즉각적 피드백을 주되, 모바일 웹 환경에서 안정적으로 동작한다.
- 음향/진동은 성능 오버헤드를 최소화하고 설정으로 사용자가 완전 제어할 수 있게 한다.
- WebView 확장 시 브릿지 교체가 최소화되도록 `audio`/`haptics` 인터페이스를 분리한다.

## 2. 지원 환경 우선순위

- 1순위: 갤럭시 S20/S21 기준 모바일 브라우저
- 2순위: 데스크톱/태블릿 브라우저(테스트 보조)
- 3순위: 향후 WebView 포장 환경

## 3. 사운드 전략(브라우저 제약 대응)

### 3.1 자동재생/권한 정책

- 페이지 로드 시 사운드는 재생하지 않는다.
- `AudioContext` 또는 첫 사용자 상호작용(`touchstart`/`click`/키 입력) 이후에만 초기화한다.
- `resume/suspend`를 통해 화면 진입/포커스 상태에 따라 음소거/재개 처리한다.
- 배경음은 기본 OFF. 추후 V2에서 옵션으로만 제공.

### 3.2 사운드 볼륨/마스터 제어

- `master`, `sfx`, `music` 구조를 둔다.
- 기본값: `master=1`, `sfx=0.7`, `music=0.3`(music는 기본 OFF이므로 미적용).
- 로컬 설정은 `localStorage`에 저장(`soundEnabled`, `sfxVolume`, `hapticsEnabled`).

## 4. 사운드 이벤트 맵

- `move` (좌/우 이동 성공)
- `rotate` (회전 성공)
- `rotate_fail` (회전 실패 또는 벽킥 실패)
- `soft_drop_tick` (소프트드롭 누적 이동 시 짧은 클릭성 톤, 과도 시 억제)
- `hard_drop` (드롭 착지/고정)
- `lock` (블록 고정)
- `line_clear_1`, `line_clear_2`, `line_clear_3`, `line_clear_4`
- `level_up`
- `game_over`
- `pause` / `resume`
- `start` / `restart`

### 4.1 사운드 믹싱 규칙

- 동일 프레임에서 `move`/`soft_drop_tick`이 과다 발생하면 1회로 데바운스(최대 60~80ms).
- 라인클리어 사운드는 다른 SFX와 동시재생 우선순위를 둔다(`line_clear_*` > `move`).
- 한 번에 8개 이상 동시 음원은 재생 제한.

## 5. 진동(Vibration) 정책

- 지원 여부 체크: `navigator.vibrate` 존재 시에만 실행.
- 진동 패턴:
  - 이동/회전: 없음
  - 하드드롭: `20ms`
  - 라인 클리어(1~2줄): `12ms`
  - 텔리스/게임오버: `20ms`
- 연쇄 입력 중 중복 진동은 프레임 기반으로 억제(최소 60ms 간격).
- WebView에서는 플랫폼 브릿지를 통해 폴백 처리 가능하도록 인터페이스 분리.

## 6. VFX 정책(시각 효과)

### 6.1 단계별 필수 VFX

- 회전 보간
  - 즉시 회전 후 0~140ms 보간 렌더
  - 과도 프레임 저하 시 자동으로 즉시 고정(보간 스킵)
- 라인 플래시
  - 클리어 라인 대상 80~120ms 깜박임
  - 다음 라인 제거/낙하 애니메이션과 동시 과부하 시 우선순위 1개만 표시
- 라인 제거 후 낙하
  - 제거 후 각 블록 그룹은 70~140ms 간격으로 하강 보간
- 하드드롭 임팩트(선택)
  - 착지 순간 `impactPulse` 알파 1회 반짝임(80~120ms)
- 오버레이 전환
  - Pause/Restart/GameOver 오버레이 140ms 페이드인/아웃

### 6.2 성능 방어선

- 동시 VFX 제한: 같은 프레임에 고해상도 이펙트 2개 초과 금지.
- FPS 저하(45fps 이하) 감지 시:
  - 라인 플래시 + 하드드롭 임팩트는 1단계 축소
  - 회전 보간은 즉시 반영

## 7. 에셋 규격

- 포맷: `wav`(짧은 효과) 또는 경량 `mp3`(호환성 확보)
- 권장 길이: 40~180ms(효과음), 1.2초 이내(기타 짧은 효과)
- 채널: 모노 1ch
- 샘플레이트: 44.1kHz
- 음량 정규화: `-16 LUFS` 근처 권장

## 8. 파일/코드 구조(예정)

- `src/services/audio.ts`
  - `init()`, `setEnabled()`, `setVolumes()`, `play(effect)`
- `src/services/vibration.ts`
  - `canVibrate()`, `pulse(ms|pattern)`, `setEnabled()`
- 효과 타입 enum 또는 상수 기반 매핑 사용.
- 실패 사유(오디오 미지원/브라우저 제한)는 조용히 무시하고 UI는 계속 동작.

## 9. 품질 기준

- 첫 조작 후 100~300ms 이내 사운드/진동 정책이 활성화된다.
- 사운드 OFF에서도 게임 피드백은 핵심 동작(점수, 게임오버, 라인클리어)이 시각적으로 유지된다.
- 배경음 OFF 상태에서 CPU 사용량이 증가하지 않는다.

## 10. 테스트 체크리스트

- 사운드 자동재생 제약: 페이지 로드 시 즉시 재생 시도 시도 없음(허용 규칙 준수).
- 사용자가 `sound OFF`를 눌렀을 때 모든 SFX가 완전 정지.
- 하드드롭/게임오버 시 진동 패턴이 1회만 발생하고 중복되지 않음.
- 라인클리어 동작 시 사운드와 플래시가 과도하게 겹치지 않음.
- 저사양 모드에서 VFX가 강제로 축소되어도 게임 가독성 저하가 크지 않음.

## 11. 향후 고도화

- 사운드 스킨(3종) 도입
- 배경 루프 트랙 추가
- 액션 타입별 다이내믹 계층화(예: 텔리스 음원 강조)
- 접근성용 화면 읽기(음성 안내) 연동 여부 검토
