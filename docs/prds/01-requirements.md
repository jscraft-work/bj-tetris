# 01-requirements: 프로젝트 요구사항

## 1. 목표 정리
- 갤럭시 S20/S21에서 최우선으로 실행 가능한 **웹 전용(Tetris 웹 앱)**을 만든다.
- 현재는 Android 네이티브 패키징 없이 브라우저에서 바로 플레이 가능한 형태로 고정한다.
- 초점은 빠른 완성(핵심 플레이), 터치 조작 안정성, 화면 대응이다.
- 출시 품질 기준: 터치 입력이 안정적이고, 프레임 드랍이 적으며, 화면 비율이 기기별로 자연스럽게 대응되어야 한다.
- 참고: 베이스는 웹으로 유지하고, 필요 시 추후 웹뷰 패키징을 통해 앱 배포 형태로 확장할 수 있다.

## 2. 기능 요구사항 (MVP)
- 플레이 기능
  - 스테이지 진행형 블록 낙하 게임
  - 기본 조작: 좌우 이동, 즉시 하드드롭, 소프트드롭, 회전
  - 블록 고정 및 라인 클리어/점수 계산
  - 게임오버/재시작
  - 일시정지/재개
- UI/UX
  - 점수, 클리어 라인, 레벨 HUD
  - 다음 블록 미리보기
  - 터치 버튼형 조작 UI(갤럭시 S20/S21 최적화)
- 운영 요소
  - 사운드 ON/OFF
  - 난이도 상승 로직

## 3. 비기능 요구사항
- 모바일 터치 반응성
  - 입력 지연 최소화(특히 회전/하드드롭)
  - 스와이프/탭 오동작 방지
- 안정성
  - 화면 회전(세로/가로)에서 레이아웃 깨짐 최소화
  - 페이지 전환/새로고침/복귀 시 상태 처리
- 성능
  - 갤럭시 S20/S21 기준으로 기본 30~60 FPS 목표
  - 불필요한 렌더 갱신 억제, 배치 렌더링 적용
- 개발 효율
  - 빠른 반복, 빠른 수정/배포
  - 테스트가 쉬운 구조

## 4. 이번 프로젝트 범위(Phase 1)
- 타깃: 모바일 웹 (갤럭시 S20/S21 기준 우선 확인)
- 조작: 화면 버튼 4종 + 스와이프 보조.
- 점수/레벨/다음 블록/일시정지/재시작 HUD.
- 사운드/진동(진동은 웹 진동 API 사용 여부 검토)
- 최소 2~3개 갤럭시 테스트 기기에서 동작 확인(S20/S21 포함)

## 5. 추후 결정해야 할 항목
- PWA 적용 여부(홈 화면 설치/오프라인 캐시)
- 분석/로그 도입 여부(게임 시작/게임오버/세션 길이)
- 사운드 정책 대응(사용자 시작 제스처 기반 재생)
