# 04-input-controls: 터치/키 입력 UX 설계

## 1. 목표

- 갤럭시 우선 환경에서 오입력 없이 빠르게 반응하는 조작을 제공한다.
- 좌우 이동, 회전, 하드드롭, 소프트드롭이 핵심이며, 조작 불일치를 최소화한다.
- 터치 우선 설계로 모바일에서 일관된 감각을 제공하고, 키보드는 보조 채널로 지원한다.

## 2. 입력 추상화

- 입력 도메인에서 실제 이벤트를 바로 소비하지 않고 `GameAction`으로 정규화한다.
- 엔진에는 아래 액션만 전달한다.

```ts
type GameAction =
  | { type: 'move_left' }
  | { type: 'move_right' }
  | { type: 'rotate_cw' }
  | { type: 'soft_drop_start' }
  | { type: 'soft_drop_end' }
  | { type: 'hard_drop' }
  | { type: 'pause_toggle' }
  | { type: 'restart' };
```

- 엔진은 매 프레임 입력 큐를 소비해 상태를 갱신한다.

## 3. 터치 버튼 UI 규칙

- 하단 고정 버튼 5개 권장 배치
  - 좌/우 이동
  - 회전
  - 소프트드롭(길게 누름)
  - 하드드롭
  - 일시정지/재개
- 최소 터치 목표 크기: `44dp` 이상(권장 56dp)
- 동작 지연 최소화를 위해 `touchstart/mousedown`에서 즉시 액션 시작, `click`만으로 제어하지 않는다.
- 버튼은 화면 끝/모서리에 배치하지 말고 손가락 접근이 쉬운 하단 중앙~좌우 영역을 사용한다.

## 4. 이동 입력 처리

- `좌/우`는 아래 규칙을 적용한다.
  - 짧은 탭: 1칸 이동
  - 길게 눌렀을 때: 반복 이동 (초기 지연 140~180ms, 이후 45~70ms)
  - 연속 입력 중 게임 보드가 막히면 즉시 중단(불필요한 충돌 시도 배제)
- 방향 전환 시에는 반복 타이머를 즉시 리셋해 오동작 방지.
- 좌우 버튼은 터치가 끝나면 즉시 이동 정지.

## 5. 회전 입력 처리

- 회전 버튼은 1회 탭당 1회 시계방향 90도 회전.
- 터치 중 떨림으로 인한 중복 이벤트는 120ms 내 중복 무시(debounce) 적용.
- 회전이 불가한 경우(벽/다른 블록 충돌)에는 별도 실패 처리 없이 무시.

## 6. 소프트드롭

- `soft_drop_start` 진입 시 즉시 소프트 속도로 낙하 모드 전환.
- 버튼 해제 시 `soft_drop_end`로 복귀.
- 장시간 누를 때 프레임별 처리로 속도만 변경하고 이동 보정은 `낙하 스텝` 단위로 수행.
- 장치별 민감도 보정용 임계값은 환경설정으로 둘 수 있다.

## 7. 하드드롭

- `hard_drop`은 즉시 현재 위치에서 최저점까지 이동 후 즉시 고정.
- 거리 기반 점수를 즉시 계산해 `score` 반영.
- 하드드롭 중 이동/회전 입력은 해당 프레임에서 무시(액션 충돌 방지).

## 8. 터치 스와이프(보조 기능)

- 터치 컨트롤 바가 닿기 어려운 상황에서 보조로 지원.
- 1회 스와이프 판정 임계값: 12~20px 이상 이동.
- 수평 스와이프는 좌/우 이동 1회로 매핑, 연속 스와이프는 연속 이동으로 처리.
- 수직 스와이프는 하단으로 짧은 거리 = 소프트드롭 시작, 길게 하락 동작 연장.
- 수직 상향 스와이프는 회전으로 바인딩하지 않는다(의도 오인식 방지).

## 9. 키보드 입력(개발자용)

- `←` 좌, `→` 우, `↑` 회전, `↓` 소프트드롭, `Space` 하드드롭, `P` pause, `R` restart
- 키보드 반복 속도는 브라우저 기본과 엔진 반복 로직을 이중 적용하지 않도록 `keydown` 상태 추적으로 일관화.
- 입력은 터치 입력과 동일한 `GameAction`으로 변환.

## 10. 입력 충돌/락 상태

- 게임 상태별 액션 허용
  - `idle`: 좌/우/회전/드롭 무시, pause/restart 허용(재시작은 idle 유지)
  - `playing`: 모든 입력 허용(단 pause/재시작 제외)
  - `paused`: 이동/회전/드롭 무시, pause 재개 허용
  - `gameover`: restart/idle 전환만 허용
- 한 프레임 내 다중 액션은 우선순위 적용
  - `pause_toggle` > `restart` > 드롭/이동/회전

## 11. 오동작 방지

- 다중 터치 처리: 기본은 단일 터치 우선(가장 최근 유효 터치), 필요 시 다중 터치는 좌우+하드드롭 조합 허용.
- 페이지 스크롤 억제: 컨트롤 영역에서 `touch-action: none`.
- 버튼 영역에서 기본 제스처(더블탭 줌, 텍스트 선택) 차단.
- `pointercancel` 대응: 입력이 중단되면 이동 반복 즉시 종료.

## 12. 피드백 정책

- 액션마다 시각 피드백: 버튼 press class 토글(크기/명도 변경).
- 사운드/진동은 입력 성공 시에만 동작.
- 하드드롭/락아웃/오류(이동 불가)에서는 불필요한 피드백 최소화.

## 13. 성능 고려

- 입력 이벤트 핸들러는 최소한으로 두고, 실제 판정은 RAF 루프에서 수행.
- 터치 이벤트가 과도하게 발생해도 상태 업데이트는 `requestAnimationFrame` 단위로 버퍼링.
- 장시간 hold 시 타이머를 재사용 풀(pool)하지 말고 단순 clearInterval/setInterval 정책으로 예측 가능하게 운영.

## 14. 구현 우선순위(실전)

1. 좌/우/회전/하드드롭/소프트드롭 터치 버튼 기본 동작
2. 입력 정규화 액션 큐 및 상태별 필터링
3. 반복 이동 및 하드드롭 즉시 고정
4. pause/restart/idle 전이
5. 스와이프 보조 추가
6. 키보드, 접근성, 오동작 방지 보정

## 15. 검증 항목

- 연속 좌우 이동 2초 이상 지속 시 블록이 끊김 없이 이동.
- 회전 탭/소프트드롭/하드드롭 동시 사용 시 충돌 없는지.
- 세로/가로 회전 상태에서 오동작 없고, 버튼 터치가 즉시 반응.
- 게임오버 직후 restart가 지연 없이 동작.
- 다중 터치와 pointercancel에 안정적으로 복구되는지.
