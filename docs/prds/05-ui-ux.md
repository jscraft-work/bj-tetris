# 05-ui-ux: 게임 화면과 HUD 설계

## 1. 목표

- 갤럭시 최적화에서 가장 중요한 목표는 **게임판 판독성**과 **터치 조작 오인식 방지**다.
- UI는 복잡한 장식보다 가독성, 버튼 접근성, 상태 피드백을 우선한다.
- 핵심 원칙:
  - 게임판이 항상 화면의 중심 시선에 있다.
  - 조작은 손가락이 닿기 쉬운 하단부로 정리한다.
  - 상태 정보(점수/라인/레벨)는 한 번에 읽히도록 상단 고정.
- Phase 1 필수 동작: 회전 애니메이션, 라인 클리어 애니메이션, 라인 낙하 애니메이션.
- 하드드롭 임팩트/오버레이 연출은 추후 고도화 항목으로 두되, 기본 안정성은 유지.

## 2. 화면 레이아웃(모바일 1 화면)

- 전체 레이아웃은 세 영역으로 나눈다.
  1. 상단 HUD 영역
  2. 중앙 게임 필드
  3. 하단 터치 조작 바
- 기본 규칙
  - 게임판 폭은 화면 너비의 82~90% 사이에서 결정.
  - 게임판은 정사각형 셀 비율 유지 (`cellSize` 동적 계산).
  - 세로 모드: 상단 HUD + 게임판 + 하단 버튼이 한 화면에 들어오게 축소.
  - 가로 모드: HUD를 좌측 상단, 게임판 중심, 버튼은 하단 또는 우측으로 재배치.
- 다음 블록 미리보기는 게임판 오른쪽 상단 또는 상단 우측 고정 박스.

## 3. HUD(상태 바)

- 항목:
  - 점수
  - 클리어 라인
  - 레벨
  - 최고 점수(선택)
  - 상태(PLAYING / PAUSED / GAME OVER)
- 표시 정책:
 - 라인/레벨 텍스트는 크고 굵게, 점수는 보조 크기로 제공.
 - 값 변경이 있을 때만 갱신하여 잦은 렌더를 줄임.
- 배치:
  - 기본: 2~3열 정렬
  - 공간이 작으면 줄바꿈 허용(가독성 유지 우선)

## 4. 게임판(필드) UI

- 보드 기본 정보
  - 10칸 x 20칸 본게임 영역
  - 2칸 상단 가드영역은 내부 로직에서만 사용, UI에는 노출하지 않음
- 선명도
  - 바둑판 선은 얇게, 블록 윤곽 대비를 높여 블록 분리를 빠르게 인지.
  - 색상 대비는 색약 대응을 고려해 채도만 올리고 밝기 변화를 둔다.
- 블록 시각
  - 고정 블록과 활성 블록은 음영/테두리 강도로 구분.
  - 하드드롭/고정 직후는 1~2프레임 강조 이펙트(선택) 적용.
- 깜박임/플리커는 피한다.

## 5. 다음 블록 미리보기(Next)

- 크기
  - 4x4 또는 4x2 미리보기 박스 권장.
  - 실제 캔버스 크기에 맞게 스케일 조정.
- 정보
 - 다음 블록 1개를 기본 표시(2개는 성능보다 가독성 우선).
- 제약
  - 미리보기가 조작을 가리지 않도록 게임판과 충분한 간격 확보.
  - 회전 상태는 `current`/`next`만 표시하지 않음.

## 6. 하단 조작 바(터치 UI)

- 필수 버튼 구성
  - 좌 / 우 이동
  - 회전
  - 소프트드롭(길게 누름)
  - 하드드롭
  - 일시정지
- 버튼 규격
  - 권장 크기: 최소 56dp
  - 탭 영역 간 간격 8~12dp(잘못 누르기 감소)
  - 상태 피드백: press 시 밝기/스케일 전환
- 동작 레이어링
  - 각 버튼은 `GameAction`을 즉시 emit
  - 소프트드롭은 `touchstart`와 `touchend`로 상태 토글
- 배치 우선순위
  - 조작 빈도가 높은 좌/우는 좌측/우측으로 분리
  - 하드드롭은 상대적으로 크고 강한 비중으로 배치
  - 회전은 좌우 사이거나 중앙 상단 쪽으로 배치

## 7. 화면 텍스트/타이포/가독성

- 텍스트 크기 원칙
  - HUD 제목: 대형(16~22sp)
  - 값(점수/라인): 중형(20~28sp)
  - 보조 텍스트: 소형(11~13sp)
- 대비
  - 배경 대비 최소 4.5:1 목표(숫자 가독성)
  - 버튼 아이콘/문구는 배경 대비 3:1 이상 유지
- 테마
  - 라이트/다크 스위칭은 1단계로 단순화하고, 기본은 고대비 라이트에 가깝게 설계.

## 8. 상태 화면(Overlay)

- 일시정지/게임오버 오버레이는 게임판 위에 반투명 레이어.
- 표시 항목:
  - 게임오버: 점수 / 라인 / 레벨 / 최고점(있으면)
  - 액션: `재시작`, `계속하기`
- 오버레이는 배경을 눌러도 게임을 조작하지 않도록 터치 차단.

## 9. 반응형 정책

- 브레이크포인트(권장)
  - 초소형: 320~375
  - 중간: 376~430
  - 가로: 431+ (가로 모드 전용 재배치)
- 리사이즈 대응
  - 회전/크기 변경 시 HUD/버튼을 다시 계산.
  - 게임 진행은 유지하되, 패딩/크기만 즉시 보정.
- safe area 대응(iOS/갤럭시 울트라): 하단에 `env(safe-area-inset-bottom)` 반영.

## 10. 접근성

- 탭/버튼 최소 크기 및 간격을 터치 테스트 기준으로 준수.
- 시각적 피드백을 반드시 추가(색 변화 + 크기 변화).
- 핵심 액션 라벨을 명시(아이콘 단독 의존 최소화).
- 사운드 자동재생은 사용자 인터랙션 이후만 허용.
- 색상만으로 의미 전달하지 않도록 텍스트/아이콘 라벨 병행.

## 11. 성능 연동 UX

- HUD·버튼은 HTML/CSS 고정 요소로 유지하고, 게임판은 Canvas로 업데이트 빈도 조절.
- 과도한 box-shadow·blur는 제거하고 단색/경계선 중심 렌더링으로 기본 품질을 유지.
- 애니메이션을 적극 반영하되, 모바일 60fps 체감 지연을 피하도록 목표를 `100~180ms` 짧은 구간으로 제한.
- 텍스트/박스는 `간헐 애니메이션`(클리어/피드백)만 사용하고, 버튼은 즉시 압축형 press feedback.

### 추천 모션 규칙

- 회전: 블록 회전 즉시 `0~140ms` 사이 완만한 보간(시계방향 기준 90도) 후 고정.
- 라인 클리어: 라인이 지워질 때 해당 라인 깜빡임 100ms + 미세 상승/붉은 엣지 강조.
- 라인 제거 후 낙하: 각 라인 단위로 블록이 `70~140ms` 간 겹치지 않게 내려오는 낙하 모션.
- 하드드롭: 최하단 도착 시 80~120ms 충돌/낙하 잔광(impact pulse)으로 피드백(선택).
- 오버레이: 게임오버/재시작 전환 140ms 페이드인.

## 12. 애니메이션 우선순위(품질/성능 균형)

- 1순위: 회전, 라인 클리어, 하드드롭 피드백
- 2순위: 오버레이/버튼 전환
- 제외(기본권장): 블록 개별 낙하 트레일, 과도한 발광, 다단 블러/필터
- 권장 규칙: 한 번에 동시에 동작하는 동적 효과는 2개 이내로 제한.

## 13. 게임 상태별 화면 변화

- `idle`: 시작 버튼/프리뷰만 노출(필요 시 안내 문구)
- `playing`: HUD + 게임판 + 조작바 모두 노출
- `paused`: 게임판 희미 처리 + resume 버튼 강조
- `gameover`: 오버레이 표시, 재시작 집중 배치

## 14. 사용자 실수 방지 UX

- 하드드롭 오인식을 막기 위해 하드드롭 버튼은 소프트드롭 버튼과 거리 분리.
- 회전과 좌우 이동이 붙어있으면 오탭 가능성이 있으므로 시각적으로 분리.
- 상하 방향 누름이 하드드롭으로 오인되지 않게 버튼 라벨 명시.

## 15. 구현 체크리스트(Phase 1)

1. 상단 HUD 4개 항목 배치 완료
2. 게임판 비율 고정 렌더(10x20) 구현
3. 다음 블록 미리보기 박스 반응형 구현
4. 하단 터치 바(4~5개) 배치 및 액션 바인딩
5. 라인 클리어 플래시(짧은 highlight) 구현
6. 일시정지/게임오버 오버레이 구현(짧은 fade-in)
7. 폰트/색 대비/버튼 크기 QA 통과

## 16. 애니메이션 품질 목표(Phase 1 필수 + 추후 항목)

- 목표: "표시가 즉시 반응 + 핵심 상태가 분명한 동작"을 위해 아래를 기본 탑재.
  - 회전 보간 애니메이션(짧은 시간)
  - 라인 클리어 시각 효과
  - 라인 제거 후 블록 낙하 애니메이션
  - 하드드롭 임팩트 플래시(선택)
- 성능 방어선:
  - 연속되는 대규모 효과가 겹칠 경우, 동일 프레임엔 **우선도가 높은 단일 효과만 렌더**.
  - 프레임 타이밍이 45fps 이하로 내려가면 동적 이펙트는 1단계 자동 축소(페이드/플래시 최소화).

## 17. 추후 고도화(Phase 2)

- 가짜 그림자(ghost piece) 미리보기
- 라인 클리어 애니메이션/배경 파편 효과
- 테마 스킨(최대 3개)
- 터치 동적 맵핑(왼손/오른손 모드)
