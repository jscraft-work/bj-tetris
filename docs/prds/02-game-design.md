# 02-game-design: 게임 규칙 및 핵심 설계

## 1. 게임 콘셉트
- 장르: 2D 블록 퍼즐(고전 테트리스 스타일)
- 목표: 일정 시간 안에 가능한 많이 블록을 안정적으로 적재해 라인을 많이 제거하고 점수를 올린다.
- 제약: 현재는 웹 전용 + 모바일 터치 중심 조작.

## 2. 보드/그리드 설계
- 기본 보드 크기: 가로 10칸, 세로 20칸
- 렌더 셀: 정사각형 픽셀 블록, 디바이스 DPR에 따라 동적 크기
- 숨겨진 상단 버퍼: 2칸
  - 스폰 영역/게임오버 판정 및 고블록 회전 충돌 처리에 사용
- 좌표계: 정수 기반 그리드 좌표 `(x, y)` 사용
- 렌더 노출 영역은 가로 10칸×세로 20칸이며, 내부 로직은 상단 가드 2칸을 포함해 22칸 높이로 처리한다.

## 3. 기본 피스(테트로미노)
- 포함 피스: I, J, L, O, S, T, Z (총 7종)
- 회전 상태: 기본 4방향(0/90/180/270), 단 O는 1개 상태로 간주 가능
- 생성 방식:
  - 기본 바구니(14-bag): 각 조각 2개씩 포함한 14개로 구성하고 셔플 후 순차 소모
  - 바구니 생성 시 낮은 확률로 초기화 토큰 추가
    - 예: 0.5% 확률로 `RESET` 토큰 1개를 바구니에 추가한 상태로 생성
    - 이때 바구니 크기는 15개가 될 수 있음
  - `RESET` 토큰이 뽑히면 즉시 현재 바구니를 버리고 바구니를 다시 생성(동일 확률 규칙으로 새 바구니 구성)
  - 현재 피스는 `active`, 다음 피스는 `next`로 미리보기
  - 회전 규칙은 기본 오프셋/시도 순서: 제자리 -> 좌/우 1칸 이동 -> 상/하 1칸 이동
- 이 규칙은 균등 분포를 유지하면서도 드문 강제 리셋 이벤트를 통해 장기 편향 체감 완화를 목표

## 4. 조작 설계
- 왼쪽/오른쪽 이동: 한 칸 단위
- 회전: 시계 방향 우선, 한 번에 90도
- 소프트드롭: 아래 방향 가속 이동
- 하드드롭: 즉시 최하단 낙하 후 즉시 고정
- 홀드(선택): 2.0 단계에서 고려
- 터치 UI(예정): 좌/우/회전/하드드롭 버튼 + 소프트드롭(길게 누르면 지속)

## 5. 게임 루프
- 프레임 루프: `requestAnimationFrame` 기반
- 낙하 타이밍:
  - 레벨에 따라 낙하 주기 감소
  - 소프트드롭은 기본 낙하 간격의 1/5~1/10 속도
  - 하드드롭은 즉시 하강 후 고정
- 라인 클리어 처리:
  - 한 번 고정 후 충돌/보드 충족 라인 스캔
  - 가득 찬 라인 제거 + 위 블록 하강
  - 연속 라인 처리 가능(동일 패턴 한 번에 여러 라인)

## 6. 점수/레벨/속도
- 점수 기준(제안)
  - 싱글 라인: 100
  - 더블: 300
  - 트리플: 500
  - 텔리스(4줄): 800
  - 백투백 보너스: 다음 클리어 시 +추가 보너스
  - 하드드롭: 하강 거리 * 2
- 레벨 계산
  - 누적 클리어 라인 기준으로 레벨업
  - 예: 10 라인당 +1 레벨 (0→1→2...)
  - 레벨 상승마다 낙하 주기 감소
- 종료 조건
  - 새 블록 스폰 위치에서 즉시 충돌하면 게임오버

## 7. 게임 상태
- 상태 값
  - `idle`(시작 전), `playing`, `paused`, `gameover`
- 핵심 데이터
  - 점수, 현재 레벨, 누적 라인, 현재/다음 피스, 보드 그리드, 드롭 속도, 소프트드롭 입력 여부
- 일시정지 규칙
  - 일시정지 시 낙하/입력 일시 정지
  - 재개 시 잔여 낙하 시간 유지/재설정 중 택1

## 8. UI/UX 규정
- 필수 요소
  - 중앙 또는 하단 우선 게임 필드
  - 상단 HUD: 점수/라인/레벨
  - 우측 하단: 다음 블록 미리보기
  - 하단 고정 버튼: 좌/우/회전/하드드롭/일시정지
- 접근성
  - 버튼은 최소 44dp 이상 터치 영역
  - 긴 터치 구간은 지연 응답(잘못 누르기 감소)
  - 사운드 자동재생은 사용자 첫 인터랙션 후 시작

## 9. 품질 기준(확정)
- 1개 라운드 내 핵심 동작(이동/회전/하드드롭)이 정상 동작해야 함
- 1분 플레이에서 크래시 없이 동작
- 라인 클리어 10개 이상이 가능한 테스트 케이스 통과
- 다양한 가로 폭에서 보드 비율이 깨지지 않아야 함

## 10. 미확정 항목(다음 단계)
- 홀드(hold), 깔끔한 가상 레이어(ghost piece), 테마/음원, 네트워크 리더보드는 3단계 이후에 반영
- 터치 우선 조작은 사용자 테스트 후 버튼 배치/반응속도 보정
